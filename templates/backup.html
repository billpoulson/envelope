Backup and restore

{% if not sqlite_ok %}

Backup and restore require a file-backed SQLite database (not :memory:). Adjust ENVELOPE_DATABASE_URL.

{% else %}

Full-database copies use SQLite’s online backup API. Passphrase-encrypted downloads use Scrypt + AES-256-GCM (same format as POST /api/v1/system/backup/database). Always use HTTPS when sending passphrases. Store ENVELOPE_MASTER_KEY separately from backup files.

Download

{% if backup_enabled %}

Treat the file like a password: it contains ciphertext, key names, and API key hashes.

Encryption uses Scrypt key derivation (CPU-heavy). Preparing the file may take 10–60 seconds—wait for the save dialog; do not double-submit (rate limit applies).

Downloads use fetch + a file blob so the response is not handled as a full-page navigation (DevTools may otherwise show type document and mis-handle the body).

{% else %}

Backup downloads are disabled (ENVELOPE_BACKUP_ENABLED=false).

{% endif %}

Restore

{% if restore_enabled %}

Replaces the entire SQLite file for this instance. Use only with a trusted backup. API: POST /api/v1/system/restore/database (multipart).

{% else %}

In-app restore is off by default. Set ENVELOPE_RESTORE_ENABLED=true to enable, or stop the service and replace the database file under the data volume manually.

{% endif %} {% endif %} {% endblock %}
