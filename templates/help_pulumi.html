Pulumi state (not the same HTTP API)

Envelope’s /tfstate/… endpoints implement the Terraform HTTP backend contract. The Pulumi CLI does not use that protocol for pulumi login — it supports backends such as PostgreSQL, S3, Azure Blob, GCS, and file.

Practical setup: run Pulumi against a supported backend, and keep the connection URL or cloud credentials in an Envelope bundle (or inject them in CI after exporting from Envelope). Example for Postgres:

# Export vars from Envelope (or CI secrets), then:
pulumi login postgres://user:pass@host:5432/pulumi

The legacy API scope name pulumi:state only applies to flat /tfstate/blobs/… keys — it does not make the stock Pulumi CLI speak Envelope’s HTTP API. Prefer Postgres/S3 state plus secrets from Envelope for Pulumi.
