Generate a recipient certificate

Step 1 of 4

This wizard helps you create an RSA key pair and a PEM certificate locally. You upload only the certificate to Envelope; the private key never leaves your machine.

  • Use a descriptive name when you register (for example laptop-alice).
  • Keep *.key files secret and backed up only in a safe store.
  • More detail: help — certificates & sealed secrets.

Where are you generating the certificate?

Run the commands for your environment. Then continue to paste the PEM into the form below.

Linux (OpenSSL)

OpenSSL is usually installed. If not: sudo apt install openssl (Debian/Ubuntu) or use your distro’s package manager.

mkdir -p ~/envelope-cert && cd ~/envelope-cert
openssl req -x509 -newkey rsa:2048 \
  -keyout recipient.key \
  -days 3650 -nodes \
  -out recipient.pem \
  -subj "/CN=envelope-recipient"
chmod 600 recipient.key
cat recipient.pem

Copy the printed -----BEGIN CERTIFICATE----- block into Certificate PEM below. Store recipient.key securely offline.

macOS (OpenSSL)

A system openssl is available on most Macs. If you use Homebrew’s OpenSSL, call the full path (for example /opt/homebrew/opt/openssl@3/bin/openssl).

mkdir -p ~/envelope-cert && cd ~/envelope-cert
openssl req -x509 -newkey rsa:2048 \
  -keyout recipient.key \
  -days 3650 -nodes \
  -out recipient.pem \
  -subj "/CN=envelope-recipient"
chmod 600 recipient.key
cat recipient.pem

Paste the certificate PEM below. Do not upload recipient.key.

Windows (OpenSSL)

Install OpenSSL, then open Git Bash or WSL and run (same as Linux). Typical installs:

  • Git for Windows — Git Bash includes OpenSSL
  • winget install ShiningLight.OpenSSL.Win64 or choco install openssl
mkdir -p ~/envelope-cert && cd ~/envelope-cert
openssl req -x509 -newkey rsa:2048 \
  -keyout recipient.key \
  -days 3650 -nodes \
  -out recipient.pem \
  -subj "/CN=envelope-recipient"
chmod 600 recipient.key
cat recipient.pem

From Command Prompt or PowerShell without Bash, either add OpenSSL to your PATH or use the PowerShell option in the previous step instead.

Windows (PowerShell)

Creates a self-signed certificate in your user store and writes envelope-recipient.pem in the current folder. Requires Windows PowerShell 5+ or PowerShell 7.

$cert = New-SelfSignedCertificate `
  -Subject "CN=envelope-recipient" `
  -DnsName "envelope-recipient.local" `
  -CertStoreLocation "Cert:\CurrentUser\My" `
  -KeyAlgorithm RSA `
  -KeyLength 2048 `
  -HashAlgorithm SHA256 `
  -NotAfter (Get-Date).AddYears(10) `
  -KeyExportPolicy Exportable `
  -KeyUsage DigitalSignature,KeyEncipherment `
  -Type Custom
$bytes = $cert.Export([System.Security.Cryptography.X509Certificates.X509ContentType]::Cert)
$b64 = [System.Convert]::ToBase64String($bytes, [System.Base64FormattingOptions]::InsertLineBreaks)
$pem = "-----BEGIN CERTIFICATE-----`r`n$b64`r`n-----END CERTIFICATE-----"
Set-Content -Path .\envelope-recipient.pem -Value $pem -Encoding ascii
Get-Content .\envelope-recipient.pem

Open envelope-recipient.pem in a text editor and paste its contents into Certificate PEM below. Optionally remove the certificate from the store when finished: Remove-Item "Cert:\CurrentUser\My\$($cert.Thumbprint)"

Enter a short Name and paste the full Certificate PEM (including BEGIN / END lines) in the form below, then Add certificate.
