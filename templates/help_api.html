Exporting bundles and stacks (API)

Single bundle

With a key that can read the bundle:

GET /api/v1/bundles/{name}/export?format=dotenv
Authorization: Bearer <api-key>

Use format=json for JSON. Requires read access to that bundle (or admin).

Stack (merged export)

GET /api/v1/stacks/{name}/export?format=dotenv
Authorization: Bearer <api-key>

The caller must be allowed to read the stack and every bundle in its layers (403 if any layer is not readable). Use format=json for JSON.

For a prefix slice (merge from the bottom through one layer only), create or use stack env-links with JSON {"through_layer_position": <n>} where n matches a layer position; omit or null for the full stack. List links with GET /api/v1/stacks/{name}/env-links to see through_layer_position and slice_label.

Opaque env URL (no Bearer on download)

Create links from the UI or POST /api/v1/bundles/{name}/env-links / POST /api/v1/stacks/{name}/env-links with a key that has write access. Download with only the path token:

curl -fsS "https://your-host/env/<token>" -o .env

Sealed secrets are not included in merged or single-bundle exports (same as typical .env export).
