Web UI

Sign in with an admin API key, then use the header to move between areas. Projects is where you pick a project to scope bundles and stacks.

  1. Sign in with an admin API key (verified on the server only).
  2. Projects — Group bundles; each project has a slug used in URLs and Terraform state paths. Open a project from the list to work with its bundles and stacks.
  3. Bundles — Create bundles under a project; each holds key/value entries.
  4. Stacks — Ordered layers of bundles; merged export applies later layers over earlier ones for duplicate keys.
  5. Variables — Edit entries on a bundle’s page; choose secret vs plaintext storage where offered.
  6. Sealed secrets — In bundle sub-nav, store ciphertext envelopes and recipient wrapped keys for client-side encrypted values (see Certificates & sealed secrets).
  7. Secret env URL — Opaque links to GET /env/<token> without putting project or bundle names in the URL. Treat links like passwords.
  8. Certificates — Under Admin → Certificates, register recipient public certificates used when uploading sealed secrets.
  9. API keys — Under Admin → API keys, create keys with scopes for CI (read, per-bundle, per-project, or admin).
